## Ce que C0 établit

Le smoke test de S1 prouvait la chaîne matérielle *via la boucle de ffmpeg*. C0 la refait **dans notre propre pipeline Rust, sur notre device, mesuré par notre horloge la plus extérieure** (§10) — c'est la baseline sur laquelle tout le reste s'empile.

- **Un seul `ID3D11Device`** (§2), créé par nous (11_1, VIDEO_SUPPORT, multithread-protected), injecté dans le décodeur D3D11VA de ffmpeg via `AVD3D11VADeviceContext.device`. Le décodeur décode donc sur *nos* textures — condition nécessaire pour que le compositeur HLSL de S3+ échantillonne directement la sortie décodeur.
- **Zéro-copie décode→encode** : `frame->hw_frames_ctx` (D3D11) passé tel quel à `h264_amf`. Seul le bitstream final descend en RAM.
- **Mesure §10 conforme** : `Instant::now()` (mappe QPC) deux fois, autour de tout le run. Rien dans la boucle. Pas de `println!`/allocation par frame. `--release` pour les chiffres publiables (ici debug pour le jalon).

## Toolchain (les pièges, pour ne pas les re-payer)

- **ffmpeg** : build **BtbN LGPL-shared 8.x** (`thirdparty/ffmpeg-master-latest-win64-lgpl-shared`), pas le gyan système (GPL). Règle K1 *et* fournit les dev libs. D3D11VA + AMF survivent au build LGPL (vérifié : `avutil gpl=false amf=true`). DLLs à mettre sur PATH au runtime (fait par `x.bat`).
- **Bindings** : générés par `bindgen` sur les headers 8.x (immunisé contre la version, vs `ffmpeg-sys-next` qui connaît 7.x). Voir `build.rs` + `wrapper.h`.
- **Structs opaques** : bindgen émet les types atteints *seulement par pointeur* (`AVFormatContext`, …) en blob opaque (`_address: u8`). Les champs `streams[]` / `pb` sont donc inaccessibles → **shim C** (`shim.c`, compilé par `cc` contre les vrais headers) qui expose des accesseurs. Offsets corrects et versionnés par construction.
- **Device lifetime** : ffmpeg `Release` le device au teardown du hwdevice ctx. On lui donne un `+1` en fuyant un clone (`std::mem::forget(gpu.device.clone())`) — sinon double-release au Drop de notre `Gpu`.
- **Build** : `x.bat <cargo-args>` = vcvars64 (INCLUDE/LIB pour libclang + linker) + ffmpeg/bin sur PATH + cargo. `.cargo/config.toml` porte `FFMPEG_DIR` et `LIBCLANG_PATH`.
