# F2.6 — Region drag snap-guide + floating tooltip

**Seed:** `tests/e2e/seed.spec.ts`
**Roadmap status:** ✅ done (`01d6b46`)

## Application Overview

The AI-edition editor lets users add **regions** to the timeline: zoom ranges, annotation overlays, and speed changes. Each region lives on its own lane below the clip track. When the user drags a region's edge (left or right handle) to resize it, the editor should:

1. **Show a vertical snap-guide line** at the new edge position so the user sees the snap target against the ruler and clip blocks.
2. **Show a floating tooltip** near the moving edge with the new `startSec → endSec` range, so the user can read the precise time without taking their hand off the mouse.

The roadblock is on `RegionTimeline.tsx`: the snap-guide and tooltip should appear during the resize gesture, then disappear when the user releases.

## Test Scenarios

### 1. Region resize — snap-guide and tooltip appear

**Setup**

1. Open the editor at `http://localhost:5173/?windowType=editor` (browser-shim mode, no Electron).
2. Apply a project with at least one asset and at least one region of any kind (zoom / annotation / speed).
3. Focus the region row in the timeline (any of: zoom lane, annotation lane, speed lane).

**Steps**

1. Locate the **left** resize handle of a region pill (the small grip on the left edge of the pill).
2. Pointer-down on the handle and drag 80px to the right without releasing.
3. While the drag is in progress, observe the lane.
4. Release the mouse.
5. Pointer-down on the **right** resize handle of a region pill and drag 60px to the left.
6. While the drag is in progress, observe the lane.

**Expected results**

- During the **left** drag, **two** vertical lines appear near the moving edge (snap guides) — one at the start of the resize, one at the projected end.
- During the **right** drag, the same two lines appear mirrored at the right edge.
- A floating tooltip appears next to the moving edge, showing the projected `startSec → endSec` in `M:SS.s` format (e.g. `0:02.1 → 0:04.5`).
- After release, the snap-guides and tooltip disappear.
- The region's actual span (read from the document state) matches the released position within ±0.1s.

**Failure signature today**

The snap-guide and tooltip DOM classes (`laneSnapGuide`, `laneDragTooltip` from `TimelinePane.module.css`) are written but never toggled by the resize handler, so the elements never appear during a drag.

### 2. Region resize — edge clamp preserves a minimum width

**Setup**

1. Same as scenario 1.
2. Pick a region of any kind.

**Steps**

1. Pointer-down on the **right** handle.
2. Drag **past** the left handle (attempting to flip the region or shrink it to zero).
3. Release.

**Expected results**

- The region's effective width is clamped to a small positive value (≥ ~50ms in source time), not zero or negative.
- The preview does not crash, freeze, or report an error.
