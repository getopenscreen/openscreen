# P1.7 — Chat panel tool-call summaries

**Seed:** `tests/e2e/seed.spec.ts`
**Roadmap status:** ✅ done (`55372df`)

## Application Overview

The chat panel in the left sidebar lets the user talk to an AI assistant. When the model calls a tool (e.g. `addSkip`, `setSkipRange`, `replaceTimeline`), the user should see a compact **"applied: <summary>"** line in the chat history under the model's message — something like `applied: trimmed 0:02.1 → 0:02.4` for a skip edit. This is the only feedback the user gets that the AI actually mutated the timeline; without it, the AI's actions are invisible.

## Test Scenarios

### 1. A tool call renders an "applied: …" line in the chat history

**Setup**

1. Open the editor at `http://localhost:5173/?windowType=editor`.
2. Open the left chat panel.
3. Configure a provider with a mock that returns a single tool call to `addSkip({ startSec: 1.0, endSec: 2.0 })`.

**Steps**

1. Type a message in the chat input ("trim a silence from 1 to 2 seconds").
2. Submit.
3. Wait for the model's response.

**Expected results**

- The chat history shows a new assistant message.
- Below the message, a compact line reads `applied: <description of the addSkip call>`. Acceptable formats include `applied: addSkip 0:01.0 → 0:02.0`, `applied: skipped 0:01.0 → 0:02.0`, or any string that mentions both time bounds.
- The line is visually distinct from the regular message (smaller, dimmer, or in a different color).

### 2. Multiple tool calls in a single response each get their own applied-line

**Setup**

1. Same as scenario 1, but the mock returns two tool calls in the same response (`addSkip` + `setClipRange`).

**Steps**

1. Send a message.
2. Wait for response.

**Expected results**

- Two applied-lines appear under the assistant message, one per tool call.
- Each describes the corresponding tool's effect.

### 3. No applied-line when the response is a regular text answer

**Setup**

1. Same as scenario 1, but the mock returns a plain text answer (no tool calls).

**Steps**

1. Send a message.
2. Wait for response.

**Expected results**

- The chat history shows the assistant message.
- **No** "applied: …" line is rendered.
- The chat panel does not show an empty applied-line slot.
